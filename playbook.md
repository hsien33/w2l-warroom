# 📕 PLAYBOOK — 財富煉金術營運治理手冊（公司的「腦」）

> **這是治理，不是操作**。能力（怎麼做）在 skills；現在到哪在 jobs.json／戰情室；歷史在 CHANGELOG。本檔記：**派工規則、分級授權、Jeff 紅線、踩過的坑、回報鐵則、各部門累積經驗**。
> 🔑 **品質保證的核心**：sub-agent 用完即散，但「經驗存這份檔、不存 agent 腦」。每個新 agent 開工先讀本檔＋自己的部門段，就繼承全部過去經驗。Jeff 退稿/讚 → 教訓寫回對應部門段 → 同錯不再犯，越用越強。

## 0. 接棒協議（每個 agent／每回合強制）
1. **醒來**：先讀 `playbook.md`（規矩）＋ `jobs.json`（在跑啥）＋認領/更新我的 job。
2. **睡前**：成果寫回 `jobs.json` → `git push`（戰情室自動刷新）→ 失敗或學到東西 → append 到本檔對應段。
3. **順序鐵則**：決定派工 → **先寫 jobs.json 再開 agent**（順序反了當機就漏記）。

## 1. 派工規則（副總用哪個機制）
- **前景 Agent**：只用在「<2 分鐘、我需當場拿結果才能決策」（查連結 200、抓錯字）。會卡住對話，能不用就不用。
- **背景 Agent（run_in_background）= 主力**：任何 >2 分鐘的單一活；Jeff 可同時繼續對話，完成才回報。
- **Workflow**：一個活要「拆成多個平行子工」才用（如全平台分發＝4 工人同時跑）。單一活別用。
- **ScheduleWakeup / Cron**：夜間自走接力＋定時巡檢殭屍案。
- **平行上限**：重活（配音/HyperFrames 渲染）一次一件；輕活（改寫/查證/排版/SEO）才放膽平行（共用 Jeff 這台機器）。

## 2. 🔴 分級授權（瓶頸＝Jeff 的核可頻寬，狂派工堆給他＝等於零）
副總天職＝**替 Jeff 過濾、自驗、打包成「一鍵 yes/no 決策包」**，用分級讓多數事不進他眼睛。
| 級 | 範圍 | 處理 |
|---|---|---|
| **A 只有 Jeff 拍板** | 對外發佈扳機、不可逆（搬站/刪除/改網域）、紅線（兒子真話/露面/投顧法宣稱/付費/帳號設定）、品牌定調（稱號/主視覺大改） | 做到「排好驗好」，**扳機 Jeff 按**；進戰情室🔴待拍板 |
| **B Jeff 快速點頭** | 金句卡/Reel/文稿/工具成品 | 做好＋附證據放戰情室🔴待拍板，Jeff 掃一眼點可發 |
| **C 副總自動做完、不打擾** | 研究/查證/改寫草稿/排版/SEO/盤點/備份/找素材/清檔 | 只記戰情室🟢完成存證，不問 Jeff |
> 待審 WIP 上限：🔴待拍板區堆太多就停派新工、先清庫存（別把 Jeff 淹死）。

## 3. 🚫 禁派清單（agent 碰不到，一律轉 A 級給 Jeff）
登入帳號的事（Hostinger/Publer/IG/FB/Threads 後台/OAuth/設 GitHub Secret/金流）、對外發佈的最後扳機、不可逆刪除。

## 4. 🔴 Jeff 紅線與硬偏好（問過一次＝永久生效，禁止重問）
- **設計**：金句卡用核可版範本、思源宋體 Black/SERIF、**金句結尾不加句號**、**真置中**、**漸層金箔不可平塗**、SS=2 超取樣、柔焦投影、冷暖中三金盤、關鍵字亮金、LOGO 微光暈不可洗掉；卡上**不放 emoji**（思源無 emoji）。親子/暖風走 [[warm-magnet-style]]（晨光奶白+斜射金+紙感+我的做法為重心+去 emoji），黑金/煉金羊皮紙是另一套別混。
- **排版**：繁中 em-dash（—）不用箭頭；中文斷行整句包 `nowrap` 只斷標點、別在詞中斷。
- **內容**：兒子對白**一律用 Jeff 真話、不編劇本**（退過 5 次）；事實正確（兒子 5A++零補習上建中；67分/老師請公假是小兒子、別點哪個別讓老師對號入座）；兒子**不露面/馬賽克**（露過面的是爸爸本人、文案「永不露面」框架照用但別說死）；好玩好笑>嚴肅、第一句別有 AI 感。
- **合規**：投顧法（不報明牌/不保證獲利/不喊跟買/觀念非投資建議/績效加警語/不誇大）。
- **通路**：Threads 連結進第一則留言、正文不放連結；連結先 curl 驗 200 才附；FB 餵無浮水印母檔走 API；戰況卡 caption 只講事實、不捏造「最/紀錄/最兇」（比較交給每日增減圖）。
- **流程**：先核後發（對外一律 Jeff 真人點頭）、不半夜亂發、不毀歷史；改母檔→build→Jeff 確認→上傳；改發文程式新檔 `git add`＋dry-run 驗雲端；工具拆導流版＋完整版；成品複製桌面成品（舊移過期/）。

## 5. 回報鐵則（Jeff 才會爽）
1. **一次給足、自主跑完、中途別回頭問**（除非卡 A 級：登入/不可逆/對外發佈）。
2. **結論先行＋附證據**（圖 inline/mp3/md5/200 截圖/抽幀），沒證據不算完成，別把 Jeff 當 QA、自己先驗最終成品本身。比草稿砍三成字。
3. **問過一次的永不再問**（寫進本檔）。
4. **我是大腦要主動**（戰況卡/轉社團/連結快過期/該固化，主動提）。
5. **守紅線、不靜默失敗（失敗要告警）、保持乾淨**（散場產出歸位、暫存清掉）。

## 6. 踩過的坑（append-only，新坑就加；詳見 `01_文件與日誌/負面案例記錄_絕不再犯.md`）
- `git commit -a` 不加新檔→雲端 ModuleNotFound（新檔要 `git add`）。
- PS 5.1 把無 BOM UTF-8 當 CP950→中文全毀（批量改 .html 用 Python 注入器，禁 PS 批次讀寫原始碼）。
- 追蹤數用瀏覽器抓非 curl（curl 是快取舊值）。
- 自動文稿捏造「衝最兇/釘首頁」＝TRUST 大忌（自動產出只講事實）。
- 兩套天數別混（集數=日期−10、戰況卡=日期−11）。
- send_message 在無人值守模式被擋＝不能自動驅動別的獨立 session（執行改用「本 session 背景派 sub-agent」）。
- 交付前驗「最終成品本身」（抽幀/抽音軌/md5），別憑改了上游就說完成（Day10 配音 6 輪教訓）。
- 🔴 給 Jeff 看成品**別用 SendUserFile（他點不開、踩過兩次：IG文稿、Day11 mp4）**→ 影音/大檔用 `Invoke-Item` 直接開檔；其餘給**桌面成品絕對路徑**或 inline 顯示（圖用 Read 秀）。
- 🔴 **別信「追蹤/進度標記檔」就下結論**（2026-06-21 兩次被金句庫「DAY003 目前發到此」帶歪，誤判 FB 金句卡死掉，其實已發到 DAY12）→ 結論前去**真實源頭驗**（FB 真貼文/IG 真卡/雲端 log），手寫的進度標記常忘了更新＝假進度。戰情室面板一律從真實源頭推導、不手寫狀態。
- 🔴 **改發文 workflow 後，確認「排程觸發不會變成真發」**（2026-06-21 fb_quote.yml 在 schedule 觸發時 `DRY_RUN=${{ inputs.dry_run }}` 為空＝會真發，差點未經核可自動發）→ 未核可的對外發佈，cron 一律先註解關閉/或強制 DRY_RUN=1；agent 宣稱「未上線」要去 yml 親驗 schedule 區塊。

## 7. 各部門累積經驗（department playbooks · 每次 Jeff 回饋就更新對應段）
> 每個部門＝一個 skill ＋ 這段累積筆記。派該部門的 agent 前，把對應段一起餵給它。

### ✍️ 文案/改寫（skill: jeff-perspective, threads/ig/reels/shorts-strategy-mentor）
- 留言/文案語感：先短認同→補自己點→謙虛收尾，比 AI 草稿砍三成字；用「的確/其實/也」少用「真的真的」；指自己貼文用「這篇」。
- 戰況卡文稿走「預告端好料」誠實調（TEASE_MODE），不白送 Email 磁鐵（孩子的第一桶金走電子報漏斗）。

### 🎨 設計/視覺（skill: 原研哉/Kelly Hoppen/Jobs/Kenya Hara + make_week.py）
- 金句卡照 make_week.py（SS2/Black/柔投影/冷暖金/漸層金箔），別自製別用 satori。
- 暖風頁照 warm-magnet-style 記憶＋範本檔。

### 📊 戰情室/數據（skill: refresh.py/index.html/build_week.py）
- 數字只放真值、缺資料標「資料待接」不假數；比較用圖表不用文案捏造。
- **「未來一周排程板」鐵則**：每格紅綠（🟢已排定/🔴未排定）一律由 `build_week.py` 從 warcard-auto 真實源頭（reels/fb_quote schedule＋cron 是否開）推導成 `week.json`，**禁止手寫狀態**；日期/星期/今天由前端 `new Date()` 即時算（別寫死星期，曾把週日寫成週六）。排程改變（補 Reel、金句卡上線、排新一週）就重跑 build_week.py。
- **治理觀**：任何「靠 Jeff 手動」的內容線＝會悄悄斷（金句卡手動 Publer 就是這樣斷掉沒人知）→ 板上恆紅提醒，能自動就自動、不能自動的每天主動提醒 Jeff。

### 🎬 影音/配音（VoxCPM + hyperframes）
- 🔴🔴🔴 **強制**：做任何「自動剪輯影片／Reel／配音」**動工前，必先讀本 playbook（尤其這 🎬 段＋多音字易錯表＋逐句聽稿驗收 SOP）**。副總派影音/剪輯 agent 時，prompt **必含一句「先讀 w2l-warroom/playbook.md 的 🎬 段再開工」**；agent 沒讀＝不准動工。（Jeff 2026-06-21 指定，避免重犯 Day11 配音錯。）
- Reel 前一天就做好；用 Jeff 真話；字級大、loudnorm I=-12。
- 🔴🔴 **配音/Reel 交付前必過「逐句聽稿核對表」**（Day11 慘案教訓 2026-06-21）：光 ffprobe 時長/解析度＋loudnorm/peak 音量＋抽幀看錯字**不算驗收**——那只是技術層。必須**逐句聽**確認：①**發音正確**（多音字查下方易錯表）②**斷句自然且與字幕一致**③**每張字幕卡都有對應配音、無啞卡**。回報要附「逐句聽稿核對結果」，不只 ffprobe。
- 🔴 **多音字易錯表（VoxCPM 常念錯；TTS 輸入用同音字替換、字幕仍顯示正字）**：血壓→TTS 餵「**寫壓**」(ㄒㄧㄝˇ)；乾脆→TTS 餵「**甘脆**」(ㄍㄢ)。（之後再發現就 append。）
- 🔴 **不准擅改核可腳本/結尾 punchline**（Day11 agent 自把結尾改成「考爛不可怕把進度握回自己手裡」＝非核可文本，且該卡還沒配音）。腳本以 Jeff 核准版逐字為準。
- 斷句技巧：每句餵 TTS 前，把「但我覺得怪：會念書的孩子…」這類易斷錯的，改寫成語意完整的句子（如「但我覺得怪的地方是，會念書的孩子，怎麼會全班一起垮」）。
